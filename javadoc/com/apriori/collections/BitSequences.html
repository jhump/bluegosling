com.apriori.collections

Class BitSequences

  • java.lang.Object
    • com.apriori.collections.BitSequences

  • public final class BitSequences
extends Object
    Utility methods for working with BitSequences.

    Most instances of BitSequence that are created by methods in this class use an array of 64-bit longs to store the sequence of bits. Exceptions include the following:

    • empty(): returns a static instance (every invocation returns the same object). This special sequence has no need for backing data.
    • singleton(boolean): uses a single boolean.
    • concat(BitSequence...): uses an array of other BitSequences. This sort of sequence may not be as efficient as an array of longs, but its performance is still good and eliminates array-copy operations which could otherwise make concatenations slower.
    • fromBitSet(BitSet): uses a BitSet to store the sequence. Under the hood, BitSet also uses an array of longs, but it does not expose the array directly so operations on a BitSet-backed BitSequence are not as efficient as those directly backed by an array.
