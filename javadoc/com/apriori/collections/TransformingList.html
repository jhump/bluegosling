    public class TransformingList<I,O>
extends TransformingCollection<I,O>
implements List<O>
    A list whose elements are the results of applying a function to another list. This list is simply a wrapper. Changes to the underlying list are visible in the transformed list. Accessing elements incurs calls to the transforming function.

    Functions that perform the transformations should be deterministic so that a stable, unchanging iterable does not appear to be mutating when accessed through this transforming wrapper.

    Since transformations can only be done in one direction, some operations are not supported. Namely, TransformingCollection.add(Object), add(int, Object), TransformingCollection.addAll(Collection), addAll(int, Collection), and set(int, Object) throw UnsupportedOperationException. Similarly, ListIterator instances returned by this list will throw exceptions if ListIterator.add(Object) or ListIterator.set(Object) is invoked.

    Also due to transformations only working in one direction, some methods are implemented in terms of the collection's transforming iterator and thus may have worse performance than the underlying collection's implementation. These methods include TransformingCollection.contains(Object), TransformingCollection.containsAll(Collection), TransformingCollection.remove(Object), TransformingCollection.removeAll(Collection), and TransformingCollection.retainAll(Collection).
