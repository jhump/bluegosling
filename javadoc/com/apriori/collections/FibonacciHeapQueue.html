Class FibonacciHeapQueue<E>

  • java.lang.Object
    • com.apriori.collections.FibonacciHeapQueue<E>
  • Type Parameters:
    E - the type of element in the queue
    All Implemented Interfaces:
    MergeablePriorityOrderedQueue<E,FibonacciHeapQueue<? extends E>>, PriorityOrderedQueue<E>, Serializable, Cloneable, Iterable<E>, Collection<E>, Queue<E>

    public class FibonacciHeapQueue<E>
extends Object
implements MergeablePriorityOrderedQueue<E,FibonacciHeapQueue<? extends E>>, Serializable, Cloneable
    A priority queue that supports merging of two queues and is backed by a Fibonacci heap. Insertion, merging, and finding the minimum (or maximum) value all run in constant time. Removing the minimum element runs in amortized logarithmic time (amortized because remove operations incur heap maintenance work, which will be logarithmic incremental work when averaged over many remove operations but can cause worst case linear performance for a single operation). Other operations, like finding or removing arbitrary elements run in linear time.

    This collection supports all optional operations of the Queue interface. Removal of an element fetched by an iterator runs in logarithmic time worst case, constant time best case.

    Iteration order for this collection is not defined other than the minimum element in the queue will be retrieved first.

      • mergeFrom

        public boolean mergeFrom(FibonacciHeapQueue<? extends E> other)
        Description copied from interface: MergeablePriorityOrderedQueue
        Merges the specified queue into this queue. The other queue will be empty when this operation completes and this queue will have the union of its own prior contents and those of the specified queue.

        This is effectively the same as the following:

         MergeablePriorityQueue<T> queue1, queue2;
 // merge queue2 into queue1
 queue1.addAll(queue2);
 queue2.clear();
        The main difference is that this specialized method can perform the merge in constant time (possibly amortized constant time). Without this method, such an operation would likely instead take linear or O(n log n) time.
