    public class TransformingSet<I,O>
extends TransformingCollection<I,O>
implements Set<O>
    A set whose elements are the results of applying a function to another collection. This set is simply a wrapper. Changes to the underlying set are visible in the transformed set. Accessing elements incurs calls to the transforming function.

    Functions that perform the transformations should be deterministic so that a stable, unchanging set does not appear to be mutating when accessed through this transforming wrapper.

    Note: The function should produce a 1-to-1 mapping from elements of the input type to elements of the output type. If the function should ever produce the same output value for two unequal input values, then the invariant that is core to the set -- that elements therein are unique -- could be broken for the transformed set. Failure to comply with this constraint may yield sets that appear to have duplicates. Because of this requirement, caution is advised when using a TransformingSet. A safer way to transform a set is to make a snapshot that is transformed and dump those into a new set which can enforce uniqueness. That could be done with code like the following:

     Set<Type1> input;
 Function<Type1, Type2> function;
 // take a snapshot to produce a transformed set that will never
 // violate set invariants
 Set<Type2> transformed = new HashSet<Type2>(
       new TransformingCollection<Type1, Type2>(input, function));

    Since transformations can only be done in one direction, some operations are not supported. Namely, TransformingCollection.add(Object) and TransformingCollection.addAll(Collection) throw UnsupportedOperationException.

    Also due to transformations only working in one direction, some methods are implemented in terms of the collection's transforming iterator and thus may have worse performance than the underlying collection's implementation. These methods include TransformingCollection.contains(Object), TransformingCollection.containsAll(Collection), TransformingCollection.remove(Object), TransformingCollection.removeAll(Collection), and TransformingCollection.retainAll(Collection).
