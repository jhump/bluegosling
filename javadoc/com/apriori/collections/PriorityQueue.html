Interface PriorityQueue<E,P>

  • Type Parameters:
    E - the type of the elements in the queue
    P - the type of the priority associated with each element
    All Superinterfaces:
    Iterable<PriorityQueue.Entry<E,P>>
    All Known Subinterfaces:
    MeldablePriorityQueue<E,P,Q>

    public interface PriorityQueue<E,P>
extends Iterable<PriorityQueue.Entry<E,P>>
    A priority queue abstract data structure. Conceptually this is like a cross between a normal Queue and a Map. Each element in the queue also has a priority, which determines the order of removal of items. It is similar to a map in that the priority of each element can be considered like a key. Unlike a map, multiple elements with the same priority are permitted (expected, actually). This type of data structure is usually backed by a heap.

    This interface varies considerably from PriorityQueue since that class does not (in fact, cannot) provide the decrease-key operation, which is a key part of the abstract data type and required by many graph traversal algorithms. For most implementations, removing an element and re-inserting with the new priority would be much much slower than just changing an element's priority using this operation.

    This class does not extend the Collection interface due to incompatibilities with the add(E, P) method's signature. Also, this container class has some API for treating it as a container of elements and other APIs for treating it as a container of entries (element + priority), a dichotomy that is not had with standard collections. If you really need a standard collection backed by a priority queue, use PriorityQueues.asQueue(PriorityQueue).

    Generally, implementations of this interface do not provide iterators that support Iterator.remove(). This is because removal of an element often has significant impact on the structure of the queue, so proceeding to iterate through subsequent items after a removal is not feasible.

    To remove an element or change its priority (as in the decrease-key operation), use methods on the PriorityQueue.Entry associated with that element.
