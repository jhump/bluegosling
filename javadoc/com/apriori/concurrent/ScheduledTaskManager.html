implements ListenableScheduledExecutorService
    An extension and enhancement to the API provided by ScheduledExecutorService.

    This implementation provides greater configurability over scheduled tasks and greater optics into the results of executions of the tasks. It is API-compatible with the standard ScheduledExecutorService, so it should behave the same if using only the standard API and not any of the extensions herein.

    Key Interfaces

    TaskDefinition
    The definition of a scheduled task. This defines when a task runs, if it is repeated and how often, what actual code is executed when the task is run, etc.
    ScheduledTaskDefinition
    A TaskDefinition that has been scheduled for execution with a ScheduledTaskManager. This provides additional API for inspecting the status of task invocations and controlling the task, like pausing/suspending executions and canceling the task.
    ScheduledTask
    A single invocation of a ScheduledTaskDefinition. If the task is defined as a repeating task, there will be multiple such invocations of this task over time. This interface is also a ScheduledFuture and is returned when submitting scheduled tasks to the executor service.
    RepeatingScheduledTask
    Represents all invocations of a repeating ScheduledTaskDefinition. This is for API and behavioral compatibility with the ScheduledFutures returned by the standard ScheduledExecutorService.scheduleAtFixedRate(Runnable, long, long, TimeUnit) and ScheduledExecutorService.scheduleWithFixedDelay(Runnable, long, long, TimeUnit) methods.

    Client code uses a TaskDefinition.Builder to construct a task and define all of the parameters for its execution. It is then scheduled with the ScheduledTaskManager for execution.

    Features

    The main features in this service, not available with the standard scheduled executor API, follow:

    • Asynchronous processing using FutureListeners. All of the futures returned by this service are instances of ListenableFuture, which has a broader and more usable API than the standard Future, including the ability to add listeners to process results asynchronously.
    • Greater control over repeated occurrences and how they are scheduled. Instead of a task either always repeating (unless an invocation fails) or never repeating (unless the logic itself schedules a successor when invoked), tasks specify a ScheduleNextTaskPolicy. Also, tasks specify a Rescheduler, which provides greater flexibility over the timing of subsequent tasks. Rescheduler implementations are provided for simple fixed-rate and fixed-delay scheduled.
    • Notification of individual invocations. For repeated tasks, instead of only being able to wait for all invocations to finish (which generally only happens after an invocation fails) or cancel all subsequent invocations, this API provides granularity at individual task level. You can listen for completions of any and all invocations. You can also cancel individual invocations of a task.
    • Greater job control. As mentioned above, you can cancel individual occurrences of a task instead of canceling the entire job. You can also pause execution of a task temporarily. This does not attempt to suspend any thread currently executing the task, but simply stops scheduling future instances of the job until the task is resumed.
    • Exception handling. You can specify an Thread.UncaughtExceptionHandler for each task to handle job failures. This, combined with a ScheduleNextTaskPolicy, provides much more flexibility in handling exceptions thrown by task invocations.
    • Job History. In addition to listening for task completions, you can access the results of recent invocations for a repeated task - the task's history.
