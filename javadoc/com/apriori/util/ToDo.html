Annotation Type ToDo


  • @Documented
@ToDo(value="Implement annotation processor that can emit TODOs during compilation, possibly as warnings. That processor should also emit errors on malformed ToDo annotations. An annotation is malformed if it has neither a value nor a tasks attribute, if it has both, or if it has ambiguous who attributes (e.g. at both ToDo level and at Task level)")
public @interface ToDo
    Describes a "TODO" task for the annotated element. Usages should either specify the value() attribute or the tasks() attribute. The latter is used when a single element has more than one associated "TODO" task.

    Examples:

     @ToDo("implement serialization and cloning")
 class MyCoolClazz {
 }
 
 @ToDo(tasks = {
    @Task("add tests!"),
    @Task("reconcile method names so API is more consistent"),
    @Task("add static helper methods for x, y, and z")
 })
 class MyCoolerClazz {
 }
