Interface Tuple

  • All Superinterfaces:
    Iterable<Object>
    All Known Subinterfaces:
    Tuple.Ops1<A>, Tuple.Ops2<A,B>, Tuple.Ops3<A,B,C>, Tuple.Ops4<A,B,C,D>, Tuple.Ops5<A,B,C,D,E>
    All Known Implementing Classes:
    AbstractTuple, Empty, NTuple, Pair, Quartet, Quintet, Trio, Unit

    public interface Tuple
extends Iterable<Object>
    Base interface implemented by all typesafe tuples. Tuples are immutable and provide a rich typesafe interface for typical operations. This interface includes methods provided by every tuple, regardless of the number of items.

    The enclosed sub-interfaces offer typesafe operations for instances with up to five items. Tuples with more than five items have less rich operations for the sixth and later items and also do not have the same level of type safety. If you need tuples with more than five items, consider using a List instead.

    See Also:
    Tuple.Ops1, Tuple.Ops2, Tuple.Ops3, Tuple.Ops4, Tuple.Ops5

      • toArray

        <T> T[] toArray(T[] a)
        Returns an array of the items in this tuple. The runtime type of the returned array is that of the specified array. If the tuple's elements fits in the specified array, they are returned therein. Otherwise, a new array is allocated with the runtime type of the specified array and the size of this collection.

        If this tuple fits in the specified array with room to spare (i.e., the array has more elements than this tuple), the element in the array immediately following the last element of the tuple is set to null. (This is useful in determining the length of this tuple only if the caller knows that this tuple does not contain any null elements.)

        Parameters:
        a - an array with the desired runtime type
        Returns:
        an array of the items in this tuple
        Throws:
        ArrayStoreException - if the runtime type of the specified array is not a supertype of the runtime type of every element in this collection
        NullPointerException - if the specified array is null
        See Also:
        Collection.toArray(Object[])
