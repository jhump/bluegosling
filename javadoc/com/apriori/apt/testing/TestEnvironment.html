com.apriori.apt.testing

Class TestEnvironment

  • java.lang.Object
    • com.apriori.apt.testing.TestEnvironment

  • public class TestEnvironment
extends Object
    Information about the current test, processing, and round environments. This represents the context of a test run by an AnnotationProcessorTestRunner. It includes references to the current ProcessingEnvironment, RoundEnvironment, JavaFileManager, etc. An instance of this class can be injected into a test method so that the test logic can interact with the environment.

      • annotationTypes

        public Set<? extends TypeElement> annotationTypes()
        Gets the set of annotation types for the current round, as TypeElements. If the test defined a processor (using @ProcessorUnderTest or @InitializeProcessorField annotations), then this will be filtered to just the annotations supported by that processor.

        If no processor is defined, this will be all annotations in the current set of classes and input files to process. If a processor is later created programmatically inside of a test method, this set can then be filtered to only the supported annotation types using filterAnnotationTypesFor(Processor).

        Returns:
        the set of annotation types for the current round of processing
        See Also:
        processorUnderTest(), filterAnnotationTypesFor(Processor)
