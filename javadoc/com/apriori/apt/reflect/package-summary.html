Package com.apriori.apt.reflect Description

This package provides an alternate set of APIs for interacting with annotations and types from an annotation processor. The layout of classes and interfaces here closely resembles those in java.lang.reflect, which many programmers will feel are more natural and easier to use than the APIs provided in the javax.lang.model.element and java.lang.model.type packages.

Most controversial among the class names here is certainly that of Class, since it "collides" with that of java.lang.Class. At the possible cost of confusion with the APIs in java.lang.reflect, these names were chosen to make the code read like reflective code. Also, since types being processed by an annotation processor typically can't be represented as runtime types (since they aren't yet compiled), code is unlikely to mix references to java.lang.reflect and com.apriori.apt.reflect, so the name shouldn't cause problems in the way of import conflicts or requiring a lot of fully-qualified type name references.

To make the most of the APIs in this package, your annotation processor should extend AbstractProcessor (instead of javax.annotation.processing.AbstractProcessor). This alternate base class provides a reference to a RoundEnvironment, which is very similar to the standard javax.annotation.processing.RoundEnvironment except that its API is defined in terms of this set of reflection APIs (instead of in terms of elements and type mirrors).

NOTE: APIs in this package requires the executing thread to be setup for the current processing environment. If you are running from the same thread on which the processor was invoked and the processor extends AbstractProcessor then this will have already been done for you.
