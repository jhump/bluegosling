com.apriori.apt.reflect

Class AnnotationBridge

  • java.lang.Object
    • com.apriori.apt.reflect.AnnotationBridge

  • public final class AnnotationBridge
extends Object
    Bridges an annotation mirror into an actual annotation interface. The trick is that classes may not be available as actual java.lang.Class tokens at runtime since they may refer to classes that are not yet compiled. So the bridge provides a way to access these class references as instances of Class instead.

    For example, take the following definition: 

     @interface AnnoXyz {
    Class<? extends Closeable>[] arrayOfClasses();
 }

    And let's say we're processing the following code: 

     @AnnoXyz(SomeClass.class, Writer.class, Reader.class)
 class SomeClass implements Closeable {
    // ...
 }
    This means that we can access the annotation in an annotation processor like so: 
     // The bridge is an instance of the AnnoXyz annotation.
 AnnoXyz annotation = createBridge(
       Class.forName("SomeClass").getAnnotation(AnnoXyz.class),
       AnnoXyz.class);
       
 // Since classes might not be available, annotation fields that return classes or
 // arrays of classes will instead return null.
 annotation.arrayOfClasses() == null; // true! 
 
 // So the bridge to the rescue!
 com.apriori.apt.reflect.Class classes[] = bridge(annotation.arrayOfClasses());
 // Yay!
 classes[0].getName(); // "SomeClass" ftw!

    So use createBridge(Annotation) to create an instance of an annotation interface based on a Annotation. And then use bridge(Class) or bridge(Class[]) to access class tokens returned from that interface. You can optionally use bridge(Object) to access other fields returned from the interface but it's not necessary.

    Since the annotation method actually returns null, the bridge(Class) method doesn't actually look at the incoming argument. It accepts an argument for syntactic sugar. In reality, the call to this method sets thread-local state to the bridged value which is then accessed via the call to bridge(Class).

      • createBridge

        public static <T extends Annotation> T createBridge(Annotation annotation,
                                    Class<T> clazz)
        Creates an annotation bridge. The returned annotation acts as would a real runtime annotation that had the same values as the specified Annotation (which itself comes from an annotation mirror).

        They key differences between an actual java.lang.annotation.Annotation and the value returned follow:

        • The returned value does not implement equals(Object) or hashCode() per the contract specified by Annotation. Bridged annotations can safely be compared to one another but not to actual (non-bridge) annotation objects.
        • Methods that return java.lang.Class or java.lang.Class[] values will always return null. Callers must use bridge(Class) or bridge(Class[]) to get to these values, but as Class objects instead of as java.lang.Class tokens.
        Parameters:
        annotation - the annotation to be bridged
        clazz - the annotation type
        Returns:
        an annotation bridge
        Throws:
        IllegalArgumentException - if the specified type is not actually an annotation type or if the specified annotation is not of the specified type
        NullPointerException - if either argument is null
