Class Holder<T>

  • java.lang.Object
    • com.apriori.possible.Holder<T>
  • Type Parameters:
    T - the type of the held value
    All Implemented Interfaces:
    Possible<T>, Serializable

    public class Holder<T>
extends Object
implements Possible<T>, Serializable
    A possible value that is mutable. The contained value can be set or cleared programmatically. So the state of whether a value is present can change. This class allows values, when present, to be null.

    This is similar to a mutable reference, not unlike AtomicReference. But this class can distinguish between a null value and one that is not present. Also note that instances of this class are not thread-safe.

      • asSet

        public Set<T> asSet()
        Returns a view of this possible value as a set. If a value is present, a singleton set with that value is returned. Otherwise, an empty set is returned.

        The returned set, being a view of this possible value, is mutable. If it is empty then the held value is absent and a call to Set.add(Object) will effectively set the held value. If a held value is already present, however, trying to add an item to the set will throw an IllegalStateException since only a single held value is permitted. If a held value is present, it can be removed from the set, effectively clearing the held value.

        As the held value is changed via calls to set(Object) or clear(), the set will also change. If the held value is changed directly through the Holder interface while a set iteration is in progress, the iterator will throw a ConcurrentModificationException. Since this class is not thread-safe, do not count on this exception for correctness. It is just a best effort to fail-fast in the case of what is clearly a programming error.

        Returns:
        a singleton set with the contained value, if present; an empty set otherwise
